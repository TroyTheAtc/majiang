# 老婆的麻将日记 — 鸿蒙端完整方案（原生）

本文档为鸿蒙**原生应用**开发方案，以 ArkTS/ArkUI 直接实现为主；本仓库 H5 已按功能拆分为与鸿蒙页面一一对应的模块，便于移植。

---

## 一、目标与阶段

| 阶段 | 目标 | 产出 |
|------|------|------|
| **主阶段** | ArkTS/ArkUI 原生实现三视图 + 本地存储 | 纯原生鸿蒙应用，可上架应用市场 |
| **可选** | 快速验证：WebView 壳加载 H5 | 可安装的 .hap，用于先跑通流程 |
| **可选** | 数据导出 / 备份 | 导出 JSON、与 Web 同步等 |

---

## 二、技术选型：鸿蒙原生方案

采用 **ArkTS + ArkUI** 原生开发，不再以 WebView 为主。

- **界面**：三个主界面（牌谱列表、搓一把表单、算账统计），使用 Tab 或 Tabs 切换，列表、表单、统计用 Column / List / 表单组件实现。
- **数据**：**首选项（Preferences）** 或 **关系型数据库** 存储记录，数据结构与本仓库 `src/js/data.js` 中的 record 格式一致（id、date、category、location、amount）。
- **资源**：将 `assets/images/` 拷贝到鸿蒙工程 `resources/base/media/`，配色与布局参考 `src/css/style.css`。

**可选**：若希望先快速在真机看到效果，可临时用 Web 组件加载本仓库 GitHub Pages 地址做验证，再按本方案做原生实现。

---

## 三、本仓库与鸿蒙模块对应关系

本仓库 `src/js/` 已按**页面与数据层**拆分，与鸿蒙工程建议一一对应：

| 本仓库（H5） | 鸿蒙对应 | 说明 |
|--------------|----------|------|
| `src/js/data.js` | `model/` + 首选项/数据库 | 存储 Key、Record 结构、getRecords/saveRecords/addRecord/updateRecord/deleteRecord、formatAmount、recordsThisYear、CATEGORIES |
| `src/js/list.js` | `pages/ListPage.ets` + 列表 ViewModel | 牌谱列表渲染、今年战绩、长按显示删除、点击编辑 |
| `src/js/add.js` | `pages/AddPage.ets` + 表单逻辑 | 新增/编辑态（editingId）、表单填充、initForm |
| `src/js/stats.js` | `pages/StatsPage.ets` + 统计逻辑 | 总战绩、胜率、按对象/地点/年份统计 |
| `src/js/view.js` | 路由 / 页面栈 | 视图切换（list / add / stats） |
| `src/js/app.js` | `EntryAbility` + 全局 UI | Tab 切换、眼睛按钮、表单提交入口、初始化 |
| `src/css/style.css` | `resources/` + ArkUI 样式 | 配色、圆角、字体、布局参考 |
| `assets/images/` | `resources/base/media/` | 背景图、图标 |

移植时可按「data → list → add → stats → 入口」顺序，逐模块对照实现。

## 四、鸿蒙工程结构建议

```
鸿蒙工程/
├── entry/src/main/
│   ├── ets/
│   │   ├── entryability/           # 入口 Ability（对应 app.js 初始化）
│   │   ├── pages/
│   │   │   ├── ListPage.ets        # 牌谱（对应 list.js）
│   │   │   ├── AddPage.ets         # 搓一把（对应 add.js）
│   │   │   └── StatsPage.ets       # 算账（对应 stats.js）
│   │   ├── model/
│   │   │   ├── Record.ets          # 记录类型（与 data.js 中结构一致）
│   │   │   └── RecordStore.ets     # 首选项/数据库读写（对应 data.js）
│   │   ├── viewmodel/              # 可选：列表/统计业务逻辑
│   │   └── common/
│   │       └── Constants.ets        # CATEGORIES、STORAGE_KEY 等
│   ├── resources/
│   │   └── base/media/             # 从 assets/images 迁入
│   └── rawfile/                    # 仅可选 WebView 验证时放 H5 资源
└── ...
```

---

## 五、实施步骤

1. **环境**：安装 DevEco Studio，创建 Empty Ability 工程，配置 HarmonyOS SDK。
2. **数据层**：参考 `src/js/data.js` 定义 Record 类型与存储 Key，实现 Preferences 或关系型数据库的读写，保证与 H5 的 JSON 结构兼容。
3. **页面**：按 List → Add → Stats 顺序实现三个页面，逻辑与 `list.js`、`add.js`、`stats.js` 对齐；资源从 `assets/images/`、`src/css/style.css` 取图与配色。
4. **入口与路由**：在 EntryAbility 或主页面中实现 Tab 切换（对应 view.js + app.js），表单提交、眼睛按钮等全局逻辑放在入口或各自页面。
5. **可选**：需要导出/备份时，增加「导出 JSON」或同步接口。

---

## 六、数据与兼容

- **数据格式**：与 H5 保持一致（id、date、category、location、amount），便于日后做 Web 与鸿蒙之间的导入/导出。
- **可选迁移**：若用户曾使用 WebView 版或 H5 版，可在首版提供「从 localStorage 导入」：通过 JS Bridge 或手动粘贴 JSON 写入鸿蒙本地存储。
