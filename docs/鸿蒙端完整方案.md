# 老婆的麻将日记 — 鸿蒙端完整方案（原生）

本文档为鸿蒙**原生应用**开发方案，以 ArkTS/ArkUI 直接实现为主。**H5 基线版本：v1.6.1**，本仓库 `src/js/` 已按功能拆分为与鸿蒙页面一一对应的模块，便于移植。

---

## 〇、开发起步（开始鸿蒙开发时按此顺序）

1. **环境**：安装 DevEco Studio，配置 HarmonyOS SDK；新建 **Empty Ability** 工程（API 9+）。
2. **资源**：将本仓库 `assets/images/` 下所有图片拷贝到鸿蒙工程 `resources/base/media/`（背景图、等级图 yishou/ping/ji/daji、yanjing、shanchu、bianji 等）。
3. **数据层**：在 `ets/` 下建 `model/Record.ets`、`model/RecordStore.ets`（或首选项封装），Key 与结构对齐 `src/js/data.js`（STORAGE_KEY、HIDE_AMOUNTS_KEY、Record：id/date/category/location/amount）。
4. **页面**：按 **List → Add → Divine → Stats** 顺序实现四个 Tab 页，逻辑分别对照 `list.js`、`add.js`、`divine.js`、`stats.js`。
5. **入口**：主界面用 Tabs 切换四个子页面，对应 `view.js` + `app.js` 的 Tab 与眼睛按钮逻辑。

## 二、技术选型：鸿蒙原生方案

采用 **ArkTS + ArkUI** 原生开发，不再以 WebView 为主。

- **界面**：四个主界面（牌谱列表、搓一把表单、占卜、算账统计），使用 Tabs 切换，列表、表单、占卜、统计用 Column / List / 表单 / 占卜布局实现。
- **数据**：**首选项（Preferences）** 或 **关系型数据库** 存储记录，数据结构与本仓库 `src/js/data.js` 一致（id、date、category、location、amount）；占卜仅读记录做「历史同期」统计，不持久化占卜结果。
- **资源**：将 `assets/images/` 拷贝到鸿蒙工程 `resources/base/media/`，配色与布局参考 `src/css/style.css`。

**可选**：若希望先快速在真机看到效果，可临时用 Web 组件加载本仓库 GitHub Pages 地址做验证，再按本方案做原生实现。

## 三、本仓库与鸿蒙模块对应关系

本仓库 `src/js/` 已按**页面与数据层**拆分，与鸿蒙工程建议一一对应（**H5 基线 v1.6.1**）：

| 本仓库（H5） | 鸿蒙对应 | 说明 |
|--------------|----------|------|
| `src/js/data.js` | `model/` + 首选项/数据库 | 存储 Key、Record 结构、getRecords/saveRecords/addRecord/updateRecord/deleteRecord、formatAmount、recordsThisYear、CATEGORIES |
| `src/js/list.js` | `pages/ListPage.ets` + 列表 ViewModel | 牌谱列表渲染、今年战绩、长按显示删除、点击编辑 |
| `src/js/add.js` | `pages/AddPage.ets` + 表单逻辑 | 新增/编辑态（editingId）、表单填充、initForm |
| `src/js/divine.js` | `pages/DivinePage.ets` + 占卜逻辑 | 日期选择、公历转农历、建除十二值日、宜忌与等级、历史同期微调、等级图与文案 |
| `src/js/stats.js` | `pages/StatsPage.ets` + 统计逻辑 | 总战绩、胜率、按对象/地点/年份统计 |
| `src/js/view.js` | 路由 / 页面栈 | 视图切换（list / add / divine / stats） |
| `src/js/app.js` | `EntryAbility` + 全局 UI | Tab 切换、眼睛按钮、表单提交入口、初始化 |
| `src/css/style.css` | `resources/` + ArkUI 样式 | 配色、圆角、字体、布局参考 |
| `assets/images/` | `resources/base/media/` | 背景图、等级图（yishou/ping/ji/daji）、图标 |

移植时可按「data → list → add → divine → stats → 入口」顺序，逐模块对照实现。

## 五、实施步骤

1. **环境**：安装 DevEco Studio，创建 Empty Ability 工程，配置 HarmonyOS SDK。
2. **数据层**：参考 `src/js/data.js` 定义 Record 类型与存储 Key，实现 Preferences 或关系型数据库的读写，保证与 H5 的 JSON 结构兼容。
3. **页面**：按 **List → Add → Divine → Stats** 顺序实现四个页面，逻辑与 `list.js`、`add.js`、`divine.js`、`stats.js` 对齐；资源从 `assets/images/`、`src/css/style.css` 取图与配色。
4. **占卜页**：对照 `divine.js` 实现公历转农历、建除十二值日（JIAN_CHU、ZHI_ORDER、JIAN_CHU_YI_JI）、almanacToLevel、历史同期 getSamePeriodStats、applySamePeriodAdjust，以及等级名/图/文案（levelNames、levelImages、levelHints）；鸿蒙无 lunar-javascript，需用 ArkTS 实现或引入鸿蒙侧农历与建除计算。
5. **入口与路由**：在主页面中实现 Tabs 切换四个子页（对应 view.js + app.js），表单提交、眼睛按钮等全局逻辑放在入口或各自页面。
6. **可选**：需要导出/备份时，增加「导出 JSON」或同步接口。

---

## 六、数据与兼容

- **数据格式**：与 H5 保持一致（id、date、category、location、amount），便于日后做 Web 与鸿蒙之间的导入/导出。
- **可选迁移**：若用户曾使用 WebView 版或 H5 版，可在首版提供「从 localStorage 导入」：通过 JS Bridge 或手动粘贴 JSON 写入鸿蒙本地存储。
